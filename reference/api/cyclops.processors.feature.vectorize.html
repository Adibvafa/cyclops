class VectorizedIndexExpression[source]ÔÉÅ

Bases: object

Functions similar to the NumPy IndexExpression for simplified slicing.

Different from the NumPy IndexExpression, it treats singular indices as ranges such that no axes are dropped, e.g., [1, 3:4, 2] is equivalent to [1:2, 3:4, 2:3]. This is necessary to keep the same axes in the same order for Vectorized objects.
